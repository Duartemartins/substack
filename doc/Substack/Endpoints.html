    module Substack::Endpoints

    The Endpoints module contains constants for all Substack API endpoints. These are used throughout the gem to make API requests.

    Most endpoints are relative to the base API URL, but some (like POSTS_FEED) are specific to a publication domain.

    Constants

    API

    Base API URL for Substack

    ATTACH_IMAGE

    Convert image URL to attachment ID for use in Notes

    FEED_FOLLOWING

    Get the user’s following feed (logged-in home feed)

    IMAGE_UPLOAD

    Binary image upload endpoint for Notes

    INBOX_SEEN

    Mark cards as read in the inbox

    INBOX_TOP

    Get the user’s inbox notifications (Notes, replies, mentions)

    LIVE_STREAMS

    Get scheduled audio rooms

    POSTS_FEED

    Get public post feed (no auth required) @param pub [String] The publication subdomain @return [String] The full API endpoint for the publication’s posts

    POST_NOTE

    Create a Note (text + attachments)

    REACT_NOTE

    React to a Note (like, etc.) @param id [String] The ID of the Note to react to @return [String] The full API endpoint for reacting to the specified Note

    UNREAD_COUNT

    Get unread message counts (DM + chat badge counter)

    USER_SETTING

    Update user settings (last_home_tab, etc.)
